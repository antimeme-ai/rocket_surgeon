# WU 1.15 TCK: Perfetto Trace Sink Behavioral Specs

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Write Gherkin behavioral specs covering the perfetto-writer wire format, RS track hierarchy, event mapping, string interning, and daemon lifecycle integration.

**Architecture:** Three `.feature` files under `tck/perfetto/` covering wire format, track hierarchy + events, and daemon lifecycle. New step definitions added to `python/tests/tck/steps/common.py`. Two new `test_*.py` modules register scenarios via pytest-bdd.

**Tech Stack:** Gherkin, pytest-bdd, existing TCK harness

---

### File Structure

| File | Purpose |
|------|---------|
| `tck/perfetto/wire-format.feature` | Protobuf wire correctness: field-1 framing, varint, packet roundtrip, interning |
| `tck/perfetto/track-hierarchy.feature` | RS→Perfetto mapping: process/rank/layer/component tracks, tick slices, probe instants |
| `tck/perfetto/daemon-lifecycle.feature` | Integration: trace created on attach, events during step, flushed on detach |
| `python/tests/tck/steps/common.py` | New step definitions for perfetto assertions |
| `python/tests/tck/test_perfetto_wire.py` | pytest-bdd scenario registration |
| `python/tests/tck/test_perfetto_tracks.py` | pytest-bdd scenario registration |
| `python/tests/tck/test_perfetto_lifecycle.py` | pytest-bdd scenario registration |
