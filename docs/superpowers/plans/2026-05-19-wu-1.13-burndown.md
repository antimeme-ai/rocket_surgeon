# WU 1.13 Burndown: Passive Hooks + CR Fixes + E2E Green

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Unblock WU 1.13 by adding passive hooks that capture container module outputs without participating in tick counting, fix all code review findings, and get the E2E test green.

**Architecture:** The existing hook infrastructure has two types: sentinel hooks (no-op, defeat PyTorch fast path) and capture hooks (mailbox barrier, tick counting). Views need container module outputs (decoder layers, self_attn, MLP) that only exist in container forward() calls. We add a third hook type — passive hooks — that stash container outputs directly into `last_outputs` without touching the mailbox/barrier/tick machinery. Passive hooks fire naturally during the forward pass after all child hooks return. Thread safety: the GIL serializes dict operations; the barrier ensures the forward thread is blocked during step-loop processing, so passive hook writes and step-loop reads never race.

**Tech Stack:** Python (passive hook installation in `bridge.py`), Rust/PyO3 (worker dispatch changes in `dispatch.rs`), Rust (adapter API to expose container paths)

```rust
pub fn resolve_with_containers(
    modules: &[RawModule],
    config: &ModelConfig,
    rank: u32,
) -> Result<(ComponentMap, Vec<String>), String> {
    let decl = family_declaration(&config.model_type)
        .ok_or_else(|| format!("unsupported model family: {}", config.model_type))?;

    let family_name = config.model_type.clone();
    let mut components = Vec::new();
    let mut vocabulary = HashSet::new();
    let mut container_paths = Vec::new();

    for module in modules {
        let mut matched = false;

        for (matcher, mapping) in &decl.mappings {
            if matches_module(matcher, module) {
                matched = true;
                match mapping {
                    ModuleMapping::Skip => break,
                    ModuleMapping::Container => {
                        container_paths.push(module.path.clone());
                        break;
                    }
                    ModuleMapping::Direct { canonical } => {
                        let layer_index = extract_layer_index(&module.path);
                        vocabulary.insert(canonical.clone());
                        let layer = layer_index.unwrap_or(0);
                        components.push(MappedComponent {
                            module_path: module.path.clone(),
                            canonical: canonical.clone(),
                            layer_index,
                            call_index: 0,
                            mapping: mapping.clone(),
                            probe_point: format!("model:{rank}:{layer}:{canonical}:0:fwd"),
                        });
                        break;
                    }
                    ModuleMapping::Fused {
                        components: fused_comps,
                    } => {
                        let layer_index = extract_layer_index(&module.path);
                        let mut resolved_fused = fused_comps.clone();
                        for fc in &mut resolved_fused {
                            if fc.split_size == 0 {
                                fc.split_size = config.hidden_size as usize;
                            }
                            vocabulary.insert(fc.canonical.clone());
                        }
                        let layer = layer_index.unwrap_or(0);
                        let fused_canonical = format!("_fused.{}", module.attr_name);
                        components.push(MappedComponent {
                            module_path: module.path.clone(),
                            canonical: fused_canonical.clone(),
                            layer_index,
                            call_index: 0,
                            mapping: ModuleMapping::Fused {
                                components: resolved_fused,
                            },
                            probe_point: format!("model:{rank}:{layer}:{fused_canonical}:0:fwd"),
                        });
                        break;
                    }
                }
            }
        }

        if !matched {
            let canonical = format!("_raw.{}", module.path);
            let layer_index = extract_layer_index(&module.path);
            let layer = layer_index.unwrap_or(0);
            vocabulary.insert(canonical.clone());
            components.push(MappedComponent {
                module_path: module.path.clone(),
                canonical: canonical.clone(),
                layer_index,
                call_index: 0,
                mapping: ModuleMapping::Direct {
                    canonical: format!("_raw.{}", module.path),
                },
                probe_point: format!("model:{rank}:{layer}:{canonical}:0:fwd"),
            });
        }
    }

    let mut vocab_sorted: Vec<String> = vocabulary.into_iter().collect();
    vocab_sorted.sort();

    Ok((
        ComponentMap {
            components,
            model_family: family_name,
            vocabulary: vocab_sorted,
        },
        container_paths,
    ))
}
```

Then rewrite `resolve()` to delegate:

```rust

    rank: u32,

}

    """Passive hooks stash container output into a dict without barriers."""

    from rocket_surgeon.bridge import (
        _models,
        install_passive_hooks,

def install_passive_hooks(
    handle: int,
    module_paths: list[str],
    storage: dict[tuple[str, int], Any],
) -> list[Any]:
    """Install plain forward hooks that stash container outputs without barriers.

    Each hook writes (path, 0) -> output into *storage*. No mailbox,
    no barrier, no tick counting. Used for views that need container
    module outputs.
    """
    model = _models[handle]
    modules_by_path = dict(model.named_modules())
    handles: list[Any] = []
    for path in module_paths:
        module = modules_by_path.get(path)
        if module is None:
            continue

        def make_hook(p: str) -> Any:
            def hook(_mod: Any, _inp: Any, output: Any) -> None:
                storage[(p, 0)] = output

            return hook

        h = module.register_forward_hook(make_hook(path))
        handles.append(h)
    return handles
```

```rust
pub fn install_passive_hooks<'py>(
    py: Python<'py>,
    handle: u64,
    container_paths: &[String],
    last_outputs: &Bound<'py, pyo3::PyAny>,
) -> anyhow::Result<Vec<pyo3::PyObject>> {
    let bridge = py.import("rocket_surgeon.bridge")?;
    let py_paths = PyList::new(py, container_paths)?;
    let result = bridge
        .getattr("install_passive_hooks")?
        .call1((handle, py_paths, last_outputs))?;
    let list = result
        .downcast::<PyList>()
        .map_err(|e| anyhow::anyhow!("expected list from install_passive_hooks, got: {e}"))?;
    let handles: Vec<pyo3::PyObject> = list
        .iter()
        .map(|h| h.into_py_any(py).unwrap())
        .collect();
    Ok(handles)
}
```

```rust
fn ensure_forward_pass(py: Python<'_>, state: &mut WorkerState, handle: u64) -> anyhow::Result<()> {
    if state.forward_pass.is_some() {
        return Ok(());
    }

    state.last_outputs = Some(PyDict::new(py).unbind().into());

    let result_mb = bridge::create_mailbox(py)?;
    let resume_mb = bridge::create_mailbox(py)?;

    let sentinel_handles = bridge::install_sentinel_hooks(handle, &state.module_paths)?;

    let (capture_handles, call_counts) = bridge::install_capture_hooks(
        py,
        handle,
        &state.module_paths,
        result_mb.bind(py),
        resume_mb.bind(py),
        &state.module_paths,
    )?;

    // Install passive hooks on container modules — these capture output
    // into last_outputs without participating in the tick/barrier protocol.
    let lo_bound = state
        .last_outputs
        .as_ref()
        .unwrap()
        .bind(py);
    let passive_handles = if state.container_paths.is_empty() {
        Vec::new()
    } else {
        bridge::install_passive_hooks(py, handle, &state.container_paths, lo_bound)?
    };

    // ... done_callback and run_forward stay the same ...

    state.forward_pass = Some(ForwardPassState {
        result_mailbox: result_mb,
        resume_mailbox: resume_mb,
        sentinel_handles,
        capture_handles,
        passive_handles,
        call_counts: call_counts.into_py_any(py)?,
        forward_complete: false,
    });
    Ok(())
}
```

## Risk Notes

1. **Thread safety of passive hooks writing to `last_outputs`**: Safe because the GIL serializes all Python dict operations. Passive hooks fire on the forward-pass thread (which holds the GIL during hook execution). The step loop's `stash_tensor_output` writes on the main thread but only during mailbox processing — the forward-pass thread is blocked on `resume_mailbox.wait()` at that point. No concurrent dict writes.

2. **Container hook ordering vs child hooks**: PyTorch fires container forward hooks AFTER the container's `forward()` returns, which is after all child module hooks have fired. This means by the time a container passive hook fires, all tick-bearing child hooks have already sent their mailbox messages. This is the correct ordering.

3. **`num_layers` stub mismatch**: The daemon's attach response returns `stub_model_info("llama")` → 32 layers. The real tiny model has 2. This is pre-existing (Issue 3 in the handoff). The E2E test works around it by not asserting `len(norms) == num_layers`. Proper fix is a separate WU.

4. **Passive hook keying with `(path, 0)`**: Using call_index 0 is correct because container modules fire exactly once per forward pass in standard (non-weight-tied) architectures. Weight-tied containers would need per-call indexing, but that's not a Phase 1 concern.
